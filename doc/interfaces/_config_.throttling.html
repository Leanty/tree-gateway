Optional delayAfter

delayAfter: number
  • Defined in config.ts:133

max number of connections during windowMs before starting to delay responses. Defaults to 1. Set to 0 to disable delaying.

Optional delayMs

delayMs: number
  • Defined in config.ts:138

milliseconds - how long to delay the response, multiplied by (number of recent hits - delayAfter). Defaults to 1000 (1 second). Set to 0 to disable delaying.

Optional headers

headers: boolean
  • Defined in config.ts:157

Enable header to show request limit and current usage.

Optional max

max: number
  • Defined in config.ts:143

max number of connections during windowMs milliseconds before sending a 429 response. Defaults to 5. Set to 0 to disable.

Optional message

message: string
  • Defined in config.ts:148

Error message returned when max is exceeded. Defaults to 'Too many requests, please try again later.'

Optional statusCode

statusCode: number
  • Defined in config.ts:153

HTTP status code returned when max is exceeded. Defaults to 429.

Optional store

store: string
  • Defined in config.ts:171

The storage to use when persisting rate limit attempts. By default, the MemoryStore is used. Possible values are:

memory

Optional windowMs

windowMs: number
  • Defined in config.ts:128
